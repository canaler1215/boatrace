このファイルは `/inner-loop` スラッシュコマンド実行中に Claude が参照する
**パラメータ調整のガイドライン**をまとめたものです。手順書は

どう調整するか」の判断基準のみを扱います。

## 重要な制約

- **バックテストは必ず GitHub Actions (`claude_fix.yml`) で実行する。ローカル実行は禁止。**
- **変更してよいのは `ml/configs/strategy_default.yaml` のみ。**
- **一度に変更するパラメータは 1 個まで。**（原因切り分けのため）

### 変更禁止パス

| パス | 理由 |
|------|------|
| `ml/src/collector/` | データ取得 — 本番データに影響 |
| `ml/src/features/` | 特徴量 — モデル入力形式が変わる |
| `ml/src/model/` | 学習・推論コード |
| `ml/migrations/`, `apps/` | DB スキーマ |
| `.github/workflows/` | CI/CD |
| `CLAUDE.md`, `AUTO_LOOP_PLAN.md`, `SETUP.md` | ドキュメント |

## パラメータ調整ガイドライン

### `prob_threshold`（確率閾値）

- 現在値: `0.07`（7%）
- 調整範囲: `0.05` 〜 `0.12`
- 1回の変更幅: `±0.01` 〜 `±0.02`
- 判断根拠:
  - 低確率帯（prob_threshold 〜 prob_threshold+0.02 の帯）の ROI が
    全体平均より −20pp 以上低い → 閾値を +0.01 or +0.02 引き上げ
  - 全確率帯の ROI が均一 → 調整しても改善余地は小さい

### `ev_threshold`（期待値閾値）

- 現在値: `2.0`
- 調整範囲: `1.5` 〜 `3.0`
- 1回の変更幅: `±0.25` 〜 `±0.5`
- 判断根拠:
  - EV 帯 `[2.0, 2.5)` の ROI が他 EV 帯より −30pp 以上低い → 2.5 に引き上げ
  - 高 EV 帯（EV ≥ 3.0）のサンプル数が 50 未満 → 閾値を上げすぎると統計的に不安定

### `exclude_courses`（除外コース）

- 現在値: `[2]`
- 1回の変更: 追加は1コースまで、削除も1コースまで
- 判断根拠:
  - **追加**: 対象コースの ROI が全体より −30pp 以上悪く、かつ 100 ベット以上ある
  - **削除**: `BET_RULE_REVIEW_202509_202512.md` で逆効果と判明した除外を戻す場合
- 注意: コース4/5 除外は過去の検証で逆効果と判明している（単純追加しない）

### `exclude_stadiums`（除外場）

- 現在値: `[11]`（びわこ）
- 1回の変更: 追加は1場まで
- 判断根拠:
  - ROI < −50% かつ ≥ 100 ベットのサンプル → 追加候補
  - 単月だけの結果で追加しない（季節性・偶然の可能性）。複数月の履歴を確認

### `min_odds`（最低オッズ）

- 現在値: `100.0`
- 調整範囲: `50` 〜 `200`
- 高オッズ戦略（宝くじ型）が現行モデルの ROI 源泉なので、大幅に下げない

### `bet_amount`, `max_bets`, `kelly_fraction`

- 基本的に触らない。ROI 改善の根本策にはならない

## 撤退条件

以下のいずれかに該当したら PR を作成せず撤退する（strategy_default.yaml は baseline に戻す）:

1. **3 イテレーション**経過しても ΔROI が +50pp 未満
2. セグメント分析で全セグメントが均等に悪い（パラメータ調整では不可、モデル再学習が必要）
3. キャリブレーション ECE が直近比 +20% 以上悪化（`run_calibration.py` で確認）
4. 変更すべき場所が `ml/configs/` 外

撤退時は以下をユーザーに報告:
- 各イテレーションで試した変更と結果
- なぜ改善できなかったか
- 推奨される次のアクション（モデル再学習 / 期間延長による再検証 など）
